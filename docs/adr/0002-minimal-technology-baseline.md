# Minimal Technology Baseline for FileVersion MDA Implementation

We will implement issues #1-#10 with a Python 3.13 FastAPI backend, Pydantic v2 validation, SQLAlchemy 2.x and Alembic over SQLite, local filesystem storage for source PDFs and report assets, PyMuPDF as the required PDF/evidence extraction engine, Chroma as a run-scoped QA index, and a React + TypeScript + Vite frontend. AnalysisRun execution will start as in-process async work coordinated by database state and cancellation checkpoints; model, vision, embedding, PDF extraction, Chroma, and asset storage integrations will sit behind thin mockable ports so deterministic CI can use fake clients. We are deliberately not introducing PostgreSQL, object storage, Redis, Celery/RQ, OCR, Docling/Marker, LangChain/LlamaIndex, Next.js, or real model/PDF service calls into the #1-#10 baseline because the current goal is to prove AnnualReport/FileVersion/MDA evidence lifecycle behavior before adding heavier infrastructure.
